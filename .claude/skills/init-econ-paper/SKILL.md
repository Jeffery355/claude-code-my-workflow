# Initialize Economics Paper

Scaffold a new empirical economics paper directory structure from `PROJECT_BRIEF.md` in the repository root.

---

## Phase 1: Verify Configuration

**Pre-condition:** `PROJECT_BRIEF.md` must exist in the repository root.

1. Read `PROJECT_BRIEF.md`
2. Extract: title, short slug, target journal, identification strategy
3. Verify all required fields are filled in (not `[placeholder]` text)
4. If any required field is missing or placeholder: stop and ask user to fill in first

**Required fields:**
- Title (not `[Paper Title]`)
- Short slug (not `[e.g., effect-minimum-wage-employment]`)
- At least one identification strategy specified

---

## Phase 2: Create Directory Structure

Create the following directory tree under `papers/[slug]/`:

```
papers/[slug]/
├── manuscript/
│   ├── [slug].tex          # Main manuscript (from template)
│   └── abstract.tex        # Abstract (extracted from manuscript)
├── tables/
│   └── README.md           # Table index
├── figures/
│   └── README.md           # Figure index
├── do-files/
│   ├── 00_master.do        # Sets globals, calls all do-files
│   ├── 01_clean.do         # Data import and cleaning
│   ├── 02_descriptives.do  # Summary statistics
│   ├── 03_regressions.do   # Main regressions
│   ├── 04_tables.do        # Table generation (esttab)
│   ├── 05_figures.do       # Figure generation
│   ├── 06_robustness.do    # Robustness checks
│   └── 07_mechanisms.do    # Mechanism analysis
├── logs/                   # Stata .log files (gitignored)
├── outputs/                # Saved results (.dta, .ster)
└── README.md               # Paper-level README
```

Create all directories using `mkdir -p`. Do not overwrite existing files.

---

## Phase 3: Populate Do-Files from Templates

For each do-file:
1. Read `templates/stata-do-file-header.do` as the base boilerplate
2. Write to `papers/[slug]/do-files/[N]_[name].do` with:
   - Header: filename, purpose, paper title, date
   - Boilerplate: `version 17`, `clear all`, `cap log close`, `set more off`
   - Global paths: from PROJECT_BRIEF.md slug
   - Log: `log using "$LOGDIR/[filename].log", text replace`
3. Leave body (after header boilerplate) with `[YOUR CODE HERE]` placeholder

For `00_master.do`, additionally:
- Set all global paths: `ROOTDIR`, `DATADIR`, `OUTDIR`, `TABLEDIR`, `FIGDIR`, `LOGDIR`
- Comment that it calls all other do-files in order
- Set `adopath++ "$DOFILEDIR/shared"` for shared utilities

---

## Phase 4: Initialize Manuscript from Template

1. Read `templates/paper-manuscript.md`
2. Replace placeholders:
   - `[Paper Title]` → actual title from PROJECT_BRIEF.md
   - `[AUTHOR: ...]` → author info
   - `[AUTHOR NAME]` → user's name (from git config or ask)
3. Write to `papers/[slug]/manuscript/[slug].tex`
4. Create `abstract.tex` separately (for conference submissions)

---

## Phase 5: Create Index README Files

**tables/README.md:**
```markdown
# Table Index: [Paper Title]

| Table | Description | Do-file |
|---|---|---|
| T1 | Summary statistics | 02_descriptives.do |
| T2 | Main results | 03_regressions.do |
| ... | ... | ... |

Update this file as you add tables.
```

**figures/README.md:**
```markdown
# Figure Index: [Paper Title]

| Figure | Description | Do-file |
|---|---|---|
| F1 | [description] | 05_figures.do |
| F2 | [description] | 05_figures.do |
| ... | ... | ... |

Update this file as you add figures.
```

**papers/[slug]/README.md:**
```markdown
# [Paper Title]

**Slug:** `[slug]`
**Target journal:** [journal]
**Identification:** [strategy]
**Created:** [YYYY-MM-DD]

## Directory Structure

- `manuscript/` — LaTeX manuscript and abstract
- `tables/` — Publication-ready regression tables (.tex)
- `figures/` — Figures generated by Stata/R
- `do-files/` — Stata do-files (run 00_master.do to execute full pipeline)
- `logs/` — Stata .log files (gitignored)
- `outputs/` — Saved regression results for reproducibility audit

## Status

**Current task:** [from PROJECT_BRIEF.md]

See `PROJECT_BRIEF.md` in the repository root for full project configuration.
```

---

## Phase 6: Verify Structure

After scaffolding, verify:
1. All directories created (no missing)
2. All do-files exist with boilerplate content
3. Manuscript .tex has been generated
4. README files created

Then report: "Paper scaffolded at `papers/[slug]/`. Next steps: (1) configure data paths in `00_master.do`, (2) start writing `01_clean.do` to import and clean your data."

---

## Important

- **Do NOT run any do-files** during scaffolding — only create the structure
- **Do NOT overwrite** existing files — check before writing
- **Use the exact slug** from PROJECT_BRIEF.md for all directory names
- **Leave bodies of do-files empty** — user fills these in as they work
